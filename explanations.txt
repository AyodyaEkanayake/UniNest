# 1️⃣ UniNest Database Structure (How Data is Connected)

Your backend currently has **5 main models**.

### 👤 User

Stores admin and student accounts.

```
User
----
_id
name
email
password
role (admin | student)
```

---

### 🏠 Hostel

```
Hostel
------
_id
name
location
totalRooms
availableRooms
facilities[]
```

Example:

```
Girls Hostel A
Near Faculty
100 rooms
WiFi, Security, Canteen
```

---

### 🚪 Room

Each hostel contains multiple rooms.

```
Room
----
_id
hostel → Hostel ID
roomNumber
capacity
occupants[]
status (Available | Full)
```

Example:

```
Room A101
Capacity: 3
Occupants: [student1, student2]
```

Relationship:

```
Hostel
   │
   ├── Room A101
   ├── Room A102
   └── Room A103
```

---

### 📝 Application

Students apply for a hostel.

```
Application
-----------
_id
student → User ID
hostel → Hostel ID
status (Pending | Approved | Rejected)
createdAt
```

Flow:

```
Student → applies → Hostel
```

---

### 🏨 Allocation

Admin assigns a **room to a student**.

```
Allocation
----------
_id
student → User ID
room → Room ID
status (Active | PendingCancellation | Cancelled)
createdAt
logs[]
```

Here is how the **whole UniNest system works**.

```
Student registers
        │
        ▼

        ▼

        │
        ▼

        │
        ▼

        │
        ▼

        │

User

 └── Allocation
         │

                │

```

# 3️⃣ How Models, Controllers, and Routes Work Together

This is **the core architecture of your backend**.

### Step 1 — Route receives request

Example:

```
POST /api/applications
```

Handled by:

```
routes/applicationRoutes.js
```

Example:

```javascript
router.post("/", verifyToken, isStudent, applicationController.applyHostel);
```

---

### Step 2 — Controller processes logic

Controller:

```
controllers/applicationController.js
```

Example:

```javascript
exports.applyHostel = async (req, res) => {

 const application = await Application.create({
   student: req.user.id,
   hostel: req.body.hostel
 });

 res.json({ message: "Application submitted" });
};
```

Controller:

* checks rules
* validates data
* talks to database

---

### Step 3 — Model communicates with MongoDB

Example:

```
models/Application.js
```

```javascript
const applicationSchema = new mongoose.Schema({
 student: { type: mongoose.Schema.Types.ObjectId, ref: "User" },
 hostel: { type: mongoose.Schema.Types.ObjectId, ref: "Hostel" }
});
```

Model = **database structure**

---

### So the flow is:

```
Client (Postman / Frontend)
          │
          ▼
Route
          │
          ▼
Controller
          │
          ▼
Model
          │
          ▼
MongoDB Database
```
